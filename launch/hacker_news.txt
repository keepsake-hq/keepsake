FIRST COMMENT (post right after submitting)

Author here. Keepsake is an open-source (Apache-2.0) long-term memory for LLMs and AI
agents that lives entirely on your machine. You own it like a crypto wallet: a 24-word
BIP-39 seed derives every key, the whole store is a single SQLCipher-encrypted file, and
there's no account and no server. Point any OpenAI-compatible client at the local proxy,
or connect an MCP agent (Claude/Cursor), and the model gets a memory that never leaves
your device unless you explicitly send it.

Why I built it: "AI memory" today mostly means your context lives on a vendor's servers,
readable by them, locked to one model, gone if they pivot. I wanted memory that's mine —
portable across models, private by construction, and erasable for real.

How it works:
- Storage: a two-plane store. Content cells are append-only ciphertext; key material lives
  in a separate, erasable key-manifest. Each cell gets a fresh *random* data key that's
  wrapped per-holder. `forget` deletes the wrapping (+ secure_delete + WAL truncation), so
  the ciphertext is unrecoverable even if someone has your seed AND an old backup — because
  the data key was never derived from the seed. That's the part most "delete" buttons fake.
- Retrieval: on-device embeddings (Nomic via ONNX) + an in-RAM vector index. Semantic, not
  keyword — "when do I fly to Berlin?" recalls "Berlin trip — arrive Friday" with no shared
  words. The index is materialized in RAM at unlock and never written in the clear.
- Surfaces: a localhost OpenAI-compatible gateway (swap your base_url) and an MCP server.
- Crypto: ML-DSA-65 (FIPS-204) signatures and an X25519 + ML-KEM-768 hybrid for sharing, so
  shared cells are post-quantum sealed. Scoped third-party access uses macaroon-style
  capability tokens (attenuable, offline-verifiable) enforced in both the proxy and MCP.
- Desktop: a Tauri v2 app (Rust core, ~15 MB) with a deliberately calm light UI; strict CSP,
  no web fonts, Tailwind compiled locally — the webview has no path to phone home.

Honest limitations (the reason I trust this post more than a marketing page):
- Zero-knowledge AT REST is real. Zero-knowledge DURING CLOUD INFERENCE is NOT — anything
  you send to a cloud model is plaintext to that provider. Keepsake makes the *local* model
  the default and turns any cloud disclosure into an explicit, minimized, logged choice
  (a per-request "privacy dial" + a signed local receipt). If a memory tool claims cloud
  inference is zero-knowledge, it's lying. I won't.
- It does NOT yet have: field-level CRDT merge (sync is state-based snapshots today), mobile,
  or an external security audit. The macOS build is unsigned/ad-hoc (more below).

I'm shipping this anonymously on purpose, so the trust has to be in the code, not in me:
Apache-2.0, no telemetry, no accounts, runs fully offline (start it with Wi-Fi off and watch
it work), and it's ~4,900 lines of Rust across 12 small crates you can read in an afternoon.
The macOS .dmg is ad-hoc signed (notarizing would tie it to a paid Apple identity), so on
first launch you right-click → Open — or just `cargo tauri build` it yourself; same result.

I'd genuinely like feedback on one decision: the erasure model trades "perfect forward
deletion" for the ability to keep your own backups — a pre-`forget` backup you personally
hold can still be opened. Is that the right line, or should there be a stricter no-backup
mode? Tear it apart.
